# ТЕХНИЧЕСКОЕ ЗАДАНИЕ

## «AI-Mentor: Turonbank» (Интеллектуальный onboarding-симулятор)

### 1. Цель проекта

* Сокращение периода адаптации (onboarding) новых сотрудников и стажёров
* Интерактивное обучение корпоративной культуре и внутренним правилам банка
* Формирование навыков работы с банковскими программами (АБС, CRM) в безопасной среде (симуляторе)
* Снижение операционной нагрузки на сотрудников HR и Учебного центра

### 2. Область применения

* Новые сотрудники (всех отделов и филиалов)
* Студенты-стажёры, пришедшие по программе «Kelajakka qadam» и другим программам
* Процессы обучения сотрудников новым программным продуктам

### 3. Основные компоненты

* AI/NLP модуль (чат-бот / эдвайзер)
* Симулятор рабочего места (UI/UX тренажёр)
* Модуль геймификации (квесты и оценивание)
* Модуль интеграции

### 4. Функциональные требования

#### 4.1 Приём и формирование данных

* Получение данных о новых сотрудниках/стажёрах из HR-системы
* Загрузка внутренних регламентов и инструкций банка в AI-базу

#### 4.2 Виртуальный AI-ассистент

* Ответы 24/7 на вопросы по банковским регламентам, дресс-коду и FAQ
* Понимание контекста и поиск точных ответов из базы документов (технология RAG)

#### 4.3 Симулятор и геймификация

* Моделирование безопасного интерфейса реальных АБС/CRM программ (dummy data)
* Выдача интерактивных заданий (обслуживание виртуального клиента) и система баллов

#### 4.4 Оценивание и принятие решений

* Progress tracking (отслеживание процента усвоения)
* Автоматическая оценка результатов и отправка данных в систему iSpring LMS

### 5. Нефункциональные требования

* Скорость: время ответа чат-бота ≤ 2 секунд
* Безопасность: отсутствие подключения к реальной клиентской базе банка, использование только искусственно сгенерированных тестовых данных (Data Leak Prevention)
* Удобство: адаптивность для мобильных устройств и веб-браузеров (Cross-platform)

### 6. Архитектура

* Frontend: React / Vue.js или Telegram Mini App
* Backend: Python (для AI/ML моделей) / Node.js
* AI Models: LLM (Large Language Model) и архитектура RAG
* API Gateway (для связи с iSpring и внутренними системами)
* PostgreSQL (для хранения прогресса пользователей и логов)

### 7. Критерии приёмки

* В MVP (первой версии) должен полностью работать как минимум 1 симулятор банковской программы и AI-чатбот
* Сокращение времени onboarding минимум на 30% в тестовой группе (стажёры)
* Успешная интеграция с системой iSpring и передача результатов
* Гарантия отсутствия использования реальных клиентских данных или коммерческой тайны банка в системе
